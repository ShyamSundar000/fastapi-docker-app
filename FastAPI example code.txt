# 🧠 Step 6: Add Real Functionality

Let’s build a **contacts API**.

---

## 📦 Step 6.1: Create Data Storage

```python
contacts = []
```

---

## ➕ Step 6.2: Add Contact (POST)

```python
from pydantic import BaseModel

class Contact(BaseModel):
    name: str
    phone: str
    email: str

@app.post("/contacts")
def add_contact(contact: Contact):
    contacts.append(contact)
    return {"message": "Contact added", "data": contact}
```

---

## 📖 Step 6.3: Get All Contacts (GET)

```python
@app.get("/contacts")
def get_contacts():
    return contacts
```

---

## 🔍 Step 6.4: Get One Contact

```python
@app.get("/contacts/{index}")
def get_contact(index: int):
    return contacts[index]
```

---

## ✏️ Step 6.5: Update Contact (PUT)

```python
@app.put("/contacts/{index}")
def update_contact(index: int, contact: Contact):
    contacts[index] = contact
    return {"message": "Updated", "data": contact}
```

---

## ❌ Step 6.6: Delete Contact

```python
@app.delete("/contacts/{index}")
def delete_contact(index: int):
    removed = contacts.pop(index)
    return {"message": "Deleted", "data": removed}
```

---

# 🎯 What You Just Learned

You now understand:

* API routes (`@app.get`, `@app.post`)
* Path parameters (`/contacts/{index}`)
* Request body (using `BaseModel`)
* CRUD operations (Create, Read, Update, Delete)
